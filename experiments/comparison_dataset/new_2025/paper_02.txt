Введение. В настоящее время вид рабочего места играет важную роль.
Многие стандартные пользовательские интерфейсы не учитывают контекст
использования пользователя, что снижает производительность труда
сотрудников. В связи с этим растет потребность в разработке динамически
изменяющегося пользовательского интерфейса, который сделает работу
комфортной и продуктивной.
Современные веб-приложения разрабатываются на основе технологий,
которые позволяют проектировать динамические пользовательские
интерфейсы, наиболее популярными из них являются: React, Vue.js, Angular.
Все инструменты разработки основаны на компонентно-ориентированном
подходе, который предполагает разделение интерфейса на независимые от
исходного кода модули программы, которые предназначены для многократного
использования. Такой подход упрощает поддержку и сопровождение
приложения, позволяет быстро адаптировать UI к различным устройствам и
меняющимся требованиям.
Наиболее популярными платформами, используемыми в высших учебных
заведениях, в настоящее время являются Moodle, 1С:Образование и Google
306
Classroom, но они обладают рядом недостатков, таких как сложность
интерфейса, ограниченные возможности и многое другое.
Для решения вышеперечисленных проблем воспользуемся методологией,
состоящей из следующих принципов. Первым из них является компонентноориентированная архитектура, а именно использование Vue.js для построения
модульных интерфейсов и разделения интерфейса на переиспользуемые
компоненты (таблицы, модальные окна, фильтры, формы). [Error! Reference
source not found.]. Второй принцип - контекстно-зависимый UI, что
подразумевает под собой адаптацию интерфейса к различным ролям
пользователей и автоматическую подстройка UI под устройство. Третий
принцип подразумевает универсальность в хранении и обработке данных, а
именно использование REST API для получения данных с сервера и
динамическое обновление интерфейса при изменении данных за счет
реактивности Vue.js. Это все соответствует принципам разделения клиентской и
серверной логики, предложенным М. Фаулером [1].
Выделим следующие этапы проектирование и разработки:
− анализ требований: определение ключевых контекстов использования,
профилей пользователей;
− проектирование устройства системы: разработка структуры
компонентов, маршрутизации Vue Router;
− разработка адаптивного пользовательского интерфейса: создание
компонентов интерфейса с помощью Vue.js, CSS с адаптацией под различные
устройства;
− работа с данными: интеграция Fetcg API для взаимодействия с
сервером, обработка данных;
− тестирование: проверка корректности, производительности и
адаптивности;
− развёртывание: docker-контейнеризация.
Оценка эффективности пользовательского интерфейса производилась по
следующим критериям: модифицируемость интерфейса, подстраиваемость под
разные устройства, персонализация под роли пользователей, удобство работы,
производительность. Сравнение характеристик разработанного интерфейса с
Moodle, 1С:Образование, Google Classroom продемонстрировало следующие
преимущества, а именно: разработанный UI характеризуется высокой
гибкостью, полной подстраиваемостью под различные устройства и
персонализацией под роли пользователей. Кроме того, он оптимизирован для
удобства работы и обеспечивает высокую производительность.
Разработанный пользовательский интерфейс обеспечивает адаптивность и
персонализацию, что делает его удобным инструментом для сотрудников
деканата. Использование Vue.js позволило создать модульную систему, которая
легко расширяется и имеет высокую производительность. Внедрение данного
интерфейса повышает эффективность работы сотрудников деканата,
обеспечивая легкий доступ к данным, а также сокращает время, необходимое
для выполнения рутинных задач.
