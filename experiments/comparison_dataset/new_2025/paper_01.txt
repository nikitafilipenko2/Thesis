Основной целью данной работы является ускорение процесса
локализации react-приложения, путем автоматизации части этапов.
Актуальность темы обусловлена растущим спросом на мультиязычные вебприложения. Локализации не только является конкурентным преимуществом
веб-приложения, но и способствует улучшению показателей поисковой
оптимизации веб-ресурса.
Существуют различные системы управления переводами вебприложения, в качестве которых могут использоваться как локальные словари,
так и облачные сервисы. Каждый подход обладает своими преимуществами и
недостатками.
При выборе системы управления переводами были выдвинуты
следующие требования: централизация, удобное администрирование,
автоматическая доставки словарей в объектное хранилище и наличие
открытого исходного кода.
В результате проведения сравнительного анализа различных решений
был выбран Tolgee – система с открытым исходным кодом, которая может быть
развернута как в собственном окружении, так и на вычислительных ресурсах
самого сервиса. Данное решение обладает рядом преимуществ: поддержка
нескольких проектов, удобный API с возможностью настройки доступа по
ключам, система авторизации с ролевой моделью, готовые настройки для
интеграции в приложение, настройки автоматической доставки переводов в
объектное хранилище, использование PostgreSQL в качестве основной базы
данных.
На предварительном этапе в исходном коде приложения, каждый
текстовый фрагмент заменяется на вызов специальной функции «t», которая
принимает в качестве входных аргументов уникальный ключ текста, значение
по умолчанию и список параметров для форматируемых текстов.
Дополнительно производится интеграция сервиса Tolgee в самом reactприложении. Для этого достаточно воспользоваться инструкцией в разделе
«Integration» административной панели сервиса.
Следующим этапом является поиск в AST (Abstract Syntax Tree) вызовов
функции «t» и извлечение переданных аргументов. AST – абстрактное
синтаксическое дерево, которое представляет каждый элемент исходного кода
(например, объявление переменной, вызов функции, условное ветвление и т.д.)
формально описывается в виде узлов. Для построения абстрактного
синтаксического дерева react-приложения используются специализированные
парсеры, одним из которых является Babel.
Дополнительно реализован экспорт извлеченных текстовых фрагментов и
уникальных ключей в CSV-файл для последующей обработки.
Сформированный на предыдущем этапе CSV-файл передается на вход другой
программе, выполняющей перевод всех текстов на выбранные языки.
Результатом работы данной программы является каталог, содержащий JSONфайлы (JavaScript Object Notation), названия которых соответствуют языковым
кодам. Эти файлы и будут используются для импорта в сервис Tolgee, который
поддерживает множество способов загрузки переводов, включая импорт JSONфайлов.
Для выполнения импорта необходимо загрузить папку с файлами словаря
на соответствующей странице в сервисе Tolgee, и выбрать целевой проект.
После подтверждения операции все ключи переводов будут добавлены в базу
данных и станут доступны для использования на сайте. Для проверки
результатов импорта достаточно обновить страницу и переключить язык
интерфейса, например, на английский.
В ходе проделанной работы был разработан метод частичной
автоматизации процесса локализации react-приложения. Предложенный подход
включает в себя разметку исходных текстовых фрагментов исходного кода
уникальными ключами, запуск скриптов для автоматизированного сбора и
перевода, а также проверку результатов для обеспечения точности перевода.
Анализ результатов показал, что после внедрения данного метода
последовало увеличение конверсии среди иностранного сегмента
пользователей веб-ресурса.
